class hydrosdk.modelversion.LocalModel(name: str, runtime: hydrosdk.image.DockerImage, payload: List[str], contract: Optional[hydro_serving_grpc.contract.model_contract_pb2.ModelContract] = None, path: Optional[str] = None, metadata: Optional[Dict[str, str]] = None, install_command: Optional[str] = None, training_data: Optional[str] = None)¶

Creates a LocalModel with all the information needed to upload it to a cluster.

Parameters

  • name (str) – The model name

  • runtime (hydrosdk.image.DockerImage) – A Docker image used to run your code.

  • payload (List[str]) – A list of paths to files (absolute or relative) with any additional resources that will be exported to the container. If the path is provided, it’ll be appended as a prefix to all paths inside the payload.

  • contract (Optional[hydro_serving_grpc.contract.model_contract_pb2.ModelContract]) – ModelContract which specifies name of function called, as well as its types and shapes of both inputs and outputs

  • path (Optional[str]) – Common path (absolute or relative) to a directory, which is used as a prefix path for all paths in a payload.

  • metadata (Optional[Dict[str, str]]) – Dictionary with string keys and values used for describing uploaded ModelVersions

  • install_command (Optional[str]) – Command to run with runtime when uploaded

  • training_data (Optional[str]) – Path (absolute, relative or an S3 URI) to a csv file with training data

class hydrosdk.modelversion.ModelVersion(id: int, model_id: int, name: str, version: int, contract: hydro_serving_grpc.contract.model_contract_pb2.ModelContract, cluster: hydrosdk.cluster.Cluster, status: Optional[hydrosdk.modelversion.ModelVersionStatus], image: Optional[Dict], runtime: Optional[hydrosdk.image.DockerImage], is_external: bool, metadata: dict = None, install_command: str = None)¶

ModelVersion is a machine learning model or a processing function that consumes provided inputs and produces predictions or transformations

ModelVersion represents one of the Model’s versions.

Parameters

  • id (int) – ModelVersion id assigned by the cluster

  • model_id (int) – Model id assigned by the cluster

  • name (str) – The name of the Model this ModelVersion belongs to.

  • version (int) – The version of the Model this ModelVersion belongs to.

  • contract (hydro_serving_grpc.contract.model_contract_pb2.ModelContract) – ModelContract which specifies name of function called, as well as its types and shapes of both inputs and outputs

  • cluster (hydrosdk.cluster.Cluster) – Hydrosphere cluster

  • status (Optional[hydrosdk.modelversion.ModelVersionStatus]) – Status of this ModelVersion, one of {Assembling, Released, Failed}

  • image (Optional[Dict]) – Dict[uri, tag, sha] which represents the ModelVersion image inside cluster DockerRegistry

  • runtime (Optional[hydrosdk.image.DockerImage]) – DockerImage of a runtime which was used to build an ‘image’

  • is_external (bool) – Indicates whether model is running outside the Cluster

  • metadata (dict) – Dictionary with string keys and values used for describing ModelVersion

  • install_command (str) – Command which was run in the runtime when this ModelVersion was uploaded
